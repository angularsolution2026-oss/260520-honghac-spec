## §1. Hypothesis (falsifiable — "Nếu X thì Y vì Z")

> **Nếu** Hero CTA primary là `Khám phá sa bàn` (variant A, đúng định vị decision platform) **thay vì** `Tìm vị trí phù hợp` (variant B, decision-focused nhưng generic hơn),
>
> **Thì** `lead_form_submit` rate trên homepage (surface `homepage_hero` + `homepage_lead_magnet` + `homepage_final_cta` cộng lại) **tăng ≥10%** sau 14 ngày,
>
> **Vì** A khớp định vị "decision platform map-first" của [00 §2](../spec/00-master-instruction.md) hơn — user click vào A sẽ navigate đến `/sa-ban` (core product), tăng engagement → tăng lead score → tăng conversion.

**Falsifiable check:** Nếu A < B hoặc bằng nhau, hypothesis bị bác bỏ.

## §3. Implementation specs

### §3.1. Assignment

- Anonymous hash visitor (Firebase Anonymous UID hashed) → bucket A hoặc B
- 50/50 split
- Sticky: 1 visitor luôn thấy cùng variant trong suốt experiment
- KHÔNG dùng PII để bucket ([04 §7](../spec/04-features-priority.md) forbidden)

### §3.2. Variant rendering

```typescript
// components/home/HeroSection.tsx
const variant = useExperiment('exp-001-hero-cta-saban-vs-vitri');

const primaryCTA = variant === 'A_saban'
  ? { label: 'Khám phá sa bàn', href: '/sa-ban' }
  : { label: 'Tìm vị trí phù hợp', href: '/sa-ban' };  // cả 2 vẫn đi /sa-ban — chỉ test copy, không test destination
```

Note: Secondary và tertiary CTA giữ nguyên cho cả 2 variants — chỉ test primary copy.

### §3.3. Tracking

GA4 event mỗi user thấy variant:
```javascript
window.dataLayer.push({
  event: 'experiment_view',
  experiment_id: 'exp-001-hero-cta-saban-vs-vitri',
  variant: 'A_saban' | 'B_vitri'
});
```

Mỗi `lead_form_submit` event include thêm parameter `experiment_variant: 'A_saban' | 'B_vitri'`.

### §3.4. Forbidden (per [04 §7](../spec/04-features-priority.md))

- ❌ Dark patterns
- ❌ Fake urgency
- ❌ PII-based assignment
- ❌ Hide consent
- ❌ Experiment trên page noindex

## §4. Decision criteria

### §4.1. Winner criteria

- A wins nếu: A `lead_form_submit` rate ≥ B × 1.10 (≥10% relative uplift) **với** p-value < 0.05
- B wins nếu: B `lead_form_submit` rate ≥ A × 1.10 với p-value < 0.05
- Inconclusive nếu: |A - B| < 10% hoặc p > 0.05 → giữ A (default theo [00 §8](../spec/00-master-instruction.md))

### §4.2. Stop conditions

- Reached `min_sample_size` + 14 ngày → analyze
- Guardrail breach > 20% (bounce rate, consent drop) → pause + investigate
- Bug phát hiện trong variant → pause + fix

### §4.3. Post-experiment

- Update `winner` field trong registry
- Update homepage code remove variant rejected
- Document `evidenceBasis` ([16 §4](../spec/16-scoring-quality-gate.md))
- Telegram digest notification

## §5. Why this experiment first?

1. **Đơn giản nhất** — chỉ thay 1 string, không thay layout
2. **Đo được trong 14 ngày** — không cần traffic khổng lồ
3. **Trả lời câu hỏi định vị quan trọng** — map-first CTA có win không?
4. **Unblock CRO 9.5 gate** — sau khi completed, có evidence cho Cat 3 ≥ 9.5
